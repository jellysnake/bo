# Research — YouTube Transcript Adapter

## Validated findings

### Rust transcript crates

Two crate paths were probed before planning implementation:

- `yt-transcript-rs` fetched video titles but failed transcript retrieval for supplied captioned videos with no captions found in the InnerTube response.
- `youtube-transcript` failed fetching/parsing transcript data for supplied captioned videos.

Decision: do not use a transcript crate for the MVP.

### Direct InnerTube player call

A direct `POST` to YouTube's InnerTube player endpoint using an Android client context returned:

- playability status
- video title
- caption track metadata
- English autogenerated caption `baseUrl` for supplied captioned videos

Fetching the selected `baseUrl` returned non-empty transcript XML.

Decision: implement a minimal internal InnerTube adapter.

## Technical unknowns to validate during implementation

### InnerTube client stability

Unknown:

- How stable the selected Android client version is.
- Whether YouTube will rate-limit or alter response structure.
- Whether some videos require a different client context.

Mitigation:

- Keep the client context isolated in `youtube.rs` constants.
- Return clear adapter errors.
- Add ignored network tests and dogfood validation, not brittle always-on network tests.

### Transcript XML variants

Observed supplied URLs returned `timedtext format="3"` with `<p>` and `<s>` nodes.
Older examples and libraries expect `<transcript><text start="...">...</text></transcript>`.

Implementation should support both:

- `timedtext` / `p` / `s` format
- simple `text` node format

Unknown:

- Whether additional XML shapes appear for manual captions.

Mitigation:

- Parse text content structurally rather than relying on one regex.
- Keep parser unit tests with representative fixtures for both known shapes.

### English language matching

The MVP is English only.

Open implementation detail:

- Whether to accept only exact `en` or also regional English codes like `en-US`, `en-GB`.

Recommended decision:

- Accept `en` and `en-*` as English.
- Prefer manual tracks before autogenerated tracks.
- Do not fall back to non-English tracks.

### Title availability

InnerTube returned titles in probes, but title should be optional in the internal model.

Mitigation:

- Fall back to video ID if title is missing or blank.

### Error taxonomy

Need useful user-facing errors without coupling YouTube failures to article-quality rejection reasons.

Recommended decision:

- Add a YouTube/adapters error variant to `CollectError`, or add a small adapter error that `CollectError` wraps.
- Avoid overloading `RejectReason::BoilerplateOnlyContent` or generic fetch errors for transcript-specific failures.

## Libraries/APIs to evaluate

### `quick-xml`

Likely useful for robust transcript XML parsing.

Pros:

- Small, focused XML parser.
- Avoids brittle regex parsing for mixed `<p><s>` text.
- Already used by some transcript libraries.

Cons:

- Adds one dependency.

Recommendation:

- Add `quick-xml` if structural XML parsing is faster/cleaner than a tiny bespoke parser.
- Do not add broader YouTube transcript crates.

### HTML/entity decoding

Transcript XML contains escaped entities such as `&#39;`.

Options:

- `quick-xml` text unescaping where sufficient.
- Add a tiny entity decoder only if needed.
- Add `html-escape` only if transcript text remains encoded after XML parsing.

Recommendation:

- First try `quick-xml` unescaping.
- Add `html-escape` only if tests show double-encoded entities remain.

## Performance considerations

- Collection is single-URL interactive work; performance constraints are modest.
- YouTube adapter requires two network requests on success:
  1. InnerTube player metadata.
  2. Transcript XML.
- Timeouts should be bounded similarly to generic fetch.
- Transcript XML can be large but should be processed streaming/event-wise or as a single string without issue for typical videos.
- Dogfood corpus runs multiple YouTube URLs; avoid excessive retries in the adapter MVP to keep runs tolerable.

## Future research/deferred improvements

- Timestamped paragraph or section source links.
- Language selection preferences.
- Replacing/changing InnerTube client context if Android becomes unreliable.
- More sophisticated transcript paragraphing.
- Fallback to browser/DOM extraction is out of scope and not planned for CLI MVP.
