# Implementation Plan — YouTube Transcript Adapter

## Architecture decisions and rationale

- Add a small internal YouTube adapter instead of using a transcript crate.
  - Crate probes showed available Rust transcript crates were either failing against the supplied captioned videos or added broader dependency/behavior surface than needed.
  - A minimal InnerTube adapter is smaller, easier to test, and keeps YouTube-specific brittleness isolated.
- Keep adapter logic outside the generic fetch/extract path.
  - YouTube videos are not article HTML; routing them through `fetch → trafilatura` is the bug this feature fixes.
  - `collect_url` should detect adapter-supported URLs before generic HTML fetch.
- Preserve existing collection semantics.
  - Duplicate detection, slug resolution, leaf writing, and index appending should be shared by ordinary pages and YouTube transcript documents.
  - Failed adapter collections must not write a leaf or index entry.
- Store no new persistent metadata.
  - YouTube transcripts become normal leaf markdown files with existing frontmatter and index fields.

## Key components and responsibilities

### `src/adapters/mod.rs`

- Defines the adapter namespace.
- Re-exports the YouTube adapter.

### `src/adapters/youtube.rs`

Responsibilities:

- Classify input URLs:
  - non-YouTube URL
  - supported YouTube video URL
  - unsupported YouTube URL with a useful reason
- Extract and validate video IDs from:
  - `youtube.com/watch?v=<video-id>`
  - `www.youtube.com/watch?v=<video-id>`
  - `youtu.be/<video-id>`
  - `youtube.com/shorts/<video-id>`
  - `www.youtube.com/shorts/<video-id>`
- Explicitly reject YouTube embed URLs.
- Fetch YouTube player metadata through InnerTube.
- Select an English caption track:
  - prefer non-ASR/manual English tracks
  - fallback to ASR/autogenerated English tracks
  - do not select non-English tracks
- Fetch transcript XML from the selected caption track.
- Parse transcript XML into plain transcript text.
- Return a transcript document containing:
  - normalized requested URL
  - video ID
  - title, falling back to video ID
  - plain markdown transcript body

### `src/collect.rs`

Responsibilities:

- Route supported YouTube video URLs to the adapter before generic fetch.
- Return graceful errors for unsupported YouTube URLs.
- Keep non-YouTube URL behavior unchanged.
- Share duplicate detection and leaf/index writing between extracted HTML content and adapter-produced content.

Expected refactor:

- Introduce a small internal write helper, e.g. `write_document(url, title, body, output_dir)`, used by both:
  - `collect_html` after extraction/quality checks
  - `collect_url` after a successful YouTube adapter fetch

### `src/lib.rs`

- Expose `adapters` if needed by tests/integration boundaries.

## Integration points and external dependencies

### Network integration

- InnerTube player endpoint:
  - `POST https://www.youtube.com/youtubei/v1/player?prettyPrint=false`
  - Android client context with `clientName: ANDROID`, `clientVersion`, and `hl: en`
  - Android-style User-Agent
- Transcript fetch:
  - `GET` selected caption track `baseUrl`

### Dependencies

- Reuse existing dependencies where possible:
  - `reqwest` for HTTP
  - `serde` / `serde_json` for partial InnerTube response decoding
  - `url` for URL parsing
- Add one XML parsing dependency only if regex/string parsing is too brittle.
  - Preferred: `quick-xml`, because transcript XML is structured and both current `timedtext` and legacy `<transcript><text>` formats are easy to parse with an event reader.
- Do not add a YouTube transcript crate for the MVP.

## Implementation strategy

1. Add adapter module skeleton.
2. Implement deterministic YouTube URL classification and video ID extraction.
3. Add unit tests for supported and unsupported URL forms.
4. Implement partial InnerTube response structs and request function.
5. Implement English caption track selection and tests.
6. Implement transcript XML parsing:
   - current `timedtext format="3"` form with `<p>` and `<s>` text nodes
   - legacy/simple `<text>` node form
   - entity decoding/whitespace normalization
7. Add transcript body formatting as plain markdown prose.
8. Refactor `collect.rs` to share the write/index path.
9. Integrate adapter routing into `collect_url` before generic HTML fetch.
10. Add collect-level tests for:
    - successful adapter-produced document write using a mocked/static adapter path where practical
    - no writes on unsupported YouTube URL or empty transcript
    - ordinary HTML collection unchanged
11. Add ignored network tests for at least one supplied captioned YouTube URL.
12. Validate:
    - `cargo fmt`
    - `cargo clippy --all-targets --all-features -- -D warnings`
    - `cargo test`
    - `./scripts/dogfood-collect video`

## Failure behavior

- Unsupported YouTube URL: fail with useful message; no leaf/index write.
- Embed URL: fail with useful message; no leaf/index write.
- No English caption track: fail with useful message; no leaf/index write.
- Caption fetch returns empty or unparsable transcript: fail with useful message; no leaf/index write.
- Network or YouTube player errors: fail with useful message; no leaf/index write.
- Non-YouTube URL: unchanged existing fetch/extract behavior.
