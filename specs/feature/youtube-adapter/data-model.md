# Data Model вЂ” YouTube Transcript Adapter

No new persistent storage is introduced. YouTube transcripts are stored as ordinary leaf markdown files and indexed through the existing `index.jsonl` format.

## Existing persisted entities

### Leaf markdown file

Existing frontmatter fields remain authoritative:

- `title`: YouTube video title, or video ID fallback
- `url`: normalized requested YouTube URL
- `collected_at`: collection timestamp
- `updated_at`: initial collection timestamp, later managed by compile

Body:

- Plain transcript markdown text.
- No timestamps.
- No transcript source links.
- No YouTube UI/page chrome.

### Index entry

Existing JSONL fields remain unchanged:

- `file`: generated markdown filename
- `title`: YouTube video title, or video ID fallback
- `url`: normalized requested YouTube URL

## Internal entities

### `YoutubeUrlMatch`

Represents URL classification before collection routing.

Fields/variants:

- `NotYoutube`
- `Supported { video_id, normalized_url }`
- `Unsupported { normalized_url, reason }`

Responsibilities:

- Prevent generic HTML collection from handling YouTube URLs that should fail explicitly.
- Preserve normalized requested URL for duplicate detection and index storage.

### `VideoId`

A validated YouTube video identifier.

Constraints:

- Non-empty.
- Accepted from supported URL forms only.
- Should reject path traversal, query-only junk, and malformed IDs.
- Conservative validation: YouTube IDs are normally 11 chars and use URL-safe characters; if implementation relaxes this, tests should still reject obvious malformed values.

### `YoutubeTranscriptDocument`

Adapter-produced document passed into the shared collection write path.

Fields:

- `url: String` вЂ” normalized requested URL
- `video_id: String`
- `title: String` вЂ” video title or video ID fallback
- `body_markdown: String` вЂ” plain transcript text

### `PlayerResponse`

Partial deserialization of the InnerTube player response. Only fields needed by this feature are modeled.

Fields:

- `video_details: Option<VideoDetails>`
- `captions: Option<Captions>`
- `playability_status: Option<PlayabilityStatus>`

### `VideoDetails`

Fields:

- `title: Option<String>`

The adapter falls back to `video_id` if `title` is absent or blank.

### `CaptionTrack`

Partial caption track model.

Fields:

- `base_url: String`
- `language_code: String`
- `kind: Option<String>` вЂ” `"asr"` means autogenerated
- `name: Option<CaptionName>` вЂ” only useful for diagnostics

Selection rules:

1. English manual track (`language_code == "en"` or starts with `"en-"`, `kind != "asr"`).
2. English autogenerated track (`kind == "asr"`).
3. Otherwise no match; do not fall back to non-English.

### `TranscriptSegment`

Parser-only representation of transcript text before formatting.

Fields:

- `text: String`

Timing data may be parsed transiently if useful for XML parsing, but it is not emitted in MVP markdown.

## Storage approach

- No schema migration.
- No new files outside the existing leaf markdown file and `index.jsonl` entry.
- Duplicate detection remains exact-match against the normalized requested URL stored in `index.jsonl`.
