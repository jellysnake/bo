### URL handling

Implemented deterministic YouTube URL classification.

Supported:

- `youtube.com/watch?v=<video-id>`
- `www.youtube.com/watch?v=<video-id>`
- `youtu.be/<video-id>`
- `youtube.com/shorts/<video-id>`
- `www.youtube.com/shorts/<video-id>`

Also supported for watch/short/shorts forms:

- extra query params like `t`, `si`, `feature`, `list`
- uppercase/lowercase host normalization through `url::Url`

Not canonicalized:

- `youtu.be/<id>` remains `https://youtu.be/<id>` in stored URL
- `watch?v=<id>` remains the original normalized requested URL

Duplicate detection remains exact-match URL based. Different URL forms for the same video can produce separate entries, matching existing `bo` semantics.

Unsupported YouTube-like URLs fail explicitly instead of falling through to ordinary HTML collection:

- embed URLs
- playlist URLs
- channel / handle URLs
- search result URLs
- mobile/music/nocookie hosts
- missing/malformed video IDs
- unsupported schemes

### InnerTube transcript fetch

Implemented synchronous `reqwest::blocking` adapter flow:

1. Classify URL and extract video ID.
2. POST InnerTube player request using Android client context.
3. Validate playability status.
4. Extract video title, falling back to video ID.
5. Select English caption track:
   - prefer manual `en` / `en-*`
   - fallback to autogenerated `en` / `en-*`
   - never select non-English or translation fallback
6. Fetch caption XML from selected `baseUrl`.
7. Parse transcript XML into plain markdown text.

### Transcript parsing

Implemented parser support for:

- current `timedtext format="3"` with `<p>` and nested `<s>` nodes
- legacy/simple `<transcript><text start="...">...</text></transcript>` shape
- text directly inside `<p>`
- nested styling tags
- blank layout paragraphs
- malformed XML rejection
- valid XML with no transcript text rejection
- XML/entity decoding
- double-encoded entity handling
- whitespace normalization

Output is plain transcript paragraphs separated by blank lines. No timestamps or source links are emitted.

### Collect integration

Updated `src/collect.rs`:

- YouTube URLs are routed to the adapter before generic HTML fetch.
- Unsupported YouTube URLs fail through `CollectError::Youtube`.
- Added shared internal write helpers for adapter-produced and HTML-extracted documents.
- Existing ordinary `collect_html` behavior is preserved.
- Failed/unsupported YouTube collection writes no leaf and no `index.jsonl`.

## Important implementation decisions

- Internal adapter instead of transcript crate.
- Synchronous `reqwest::blocking` to match existing collect pipeline.
- `quick-xml` for XML parsing.
- Store normalized user-requested URL, not canonical video URL.
- Exact-match duplicate semantics remain unchanged.
- English means `en` or `en-*`.
- No translation fallback.
- Manual captions preferred over autogenerated captions.
- YouTube failures use `CollectError::Youtube` rather than `RejectReason`.

## Known limitations / future work

- InnerTube is unofficial and may change.
- Network/dogfood success depends on YouTube availability, IP reputation, and region.
- Same video collected via different URL forms creates separate entries.
- No timestamp/source links in output.
- No language selection.
- No transcript quality filtering beyond empty/unparseable rejection.
- No playlist/channel/embed extraction.

Future improvement already noted in `scratchpad/features.md`:

- optional paragraph/section timestamps or source links
- revisit dependency posture if a smaller internal parser becomes worthwhile
