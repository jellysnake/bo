# YouTube Transcript Adapter

## Problem statement

`bo collect` currently treats YouTube video URLs like ordinary web pages. That produces poor results because the useful collectable text for a video is usually its transcript, not the YouTube watch page UI.

Users should be able to collect supported YouTube video URLs and receive a normal markdown leaf containing the English transcript. If a transcript cannot be collected, `bo` should fail explicitly without writing junk.

## User-facing requirements

- A user can run `bo collect <url>` for these YouTube video URL forms:
  - `https://www.youtube.com/watch?v=<video-id>`
  - `https://youtube.com/watch?v=<video-id>`
  - `https://youtu.be/<video-id>`
  - `https://www.youtube.com/shorts/<video-id>`
  - `https://youtube.com/shorts/<video-id>`
- For supported YouTube video URLs, `bo` collects the English transcript as the leaf body.
- Autogenerated English captions are acceptable when manually authored English captions are not available.
- The generated markdown body is plain transcript text without timestamps, citation links, or per-line source references.
- The leaf title is the YouTube video title when available.
- If the YouTube video title is unavailable, the leaf title falls back to the video ID.
- The collected URL recorded in the index is the user-requested YouTube video URL in normalized form.
- If the URL is a YouTube URL but not a supported video URL, `bo` fails gracefully with a useful reason.
- YouTube embed URLs are not collected by this adapter and should fail gracefully.
- If English captions/transcript data are unavailable, disabled, inaccessible, or empty, `bo` fails explicitly without writing a leaf file or index entry.
- Duplicate detection works for collected YouTube URLs the same way it works for ordinary collected pages.

## Success criteria

- Collecting a supported YouTube watch URL with English captions creates one markdown leaf and one index entry.
- Collecting a supported `youtu.be` URL with English captions creates one markdown leaf and one index entry.
- Collecting a supported YouTube Shorts URL with English captions creates one markdown leaf and one index entry.
- The leaf body contains readable transcript prose and does not contain YouTube page chrome.
- The leaf body does not include timestamps or transcript source links.
- The leaf title uses the video title when it can be determined.
- The leaf title falls back to the video ID when the video title cannot be determined.
- A supported YouTube video URL with no available English transcript is rejected without file/index writes.
- A YouTube embed URL is rejected without file/index writes.
- A malformed or non-video YouTube URL is rejected without file/index writes.
- Existing ordinary web page collection behavior remains unchanged.
- The video dogfood corpus can be used to validate transcript collection against a set of known captioned videos.

## Out of scope

- Non-English transcript selection or translation.
- User-configurable transcript language preferences.
- Timestamped transcript output.
- Per-paragraph or per-section source links into the video.
- Playlist, channel, search result, or comment collection.
- Collecting embedded videos from arbitrary web pages.
- Speaker identification or diarization.
- Summarization or transcript cleanup beyond producing readable plain transcript markdown.
- Guaranteeing transcript availability for every public YouTube video.
