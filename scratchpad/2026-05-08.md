### Why this shape

- **domain/** is the gravity well. Every layer depends on it; it depends on
  nothing. These aren't ORM "models" вҖ” they encode the tree/branch/leaf metaphor
  that IS the product. Pure data + invariants + transformations.

- **engine/** does singular jobs: fetch HTML, extract content, run the agent loop.
  It's the layer that crosses the network boundary by consuming adapters.

- **cli/** orchestrates engine modules into user-facing pipelines. One file per
  subcommand. Thin glue, no business logic of its own.

- **adapters/** encapsulates protocol-specific knowledge (innertube, transcript
  parsing). Consumed only by engine. New source types (RSS, Pocket export, etc.)
  land here without touching anything above.

- Dependencies flow strictly downward. No upward or lateral arrows between layers.
  domain never knows adapters exist. cli never imports from adapters directly.
