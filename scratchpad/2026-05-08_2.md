# 2026-05-08 (2)

## The compile agent loop is the wrong mechanism

Bo's current `compile` command uses an internal agent loop: the LLM decides which
tools to call, in what order, until it judges the task done. That's the wrong
abstraction here because the workflow is completely predetermined:

```
list index → read all leaves → identify concepts → write branches → update frontmatter
```

The LLM never needs to decide "what to do next." Only steps 3–4 require LLM
creativity. Everything else is deterministic code.

## The correct pattern

> Deterministic pipeline with a single structured LLM call in the middle.

```
code gathers context  →  one structured-output LLM call  →  code writes results
```

No loop. No tool use by the model. The model receives a well-formed prompt with all
necessary context already assembled, returns structured output (JSON), and code
handles all I/O on both sides.

## Applied to all planned LLM-powered commands

| Command | Code (before) | LLM (1 call) | Code (after) |
|---|---|---|---|
| `bo compile` | read all leaves | find patterns / synthesize concepts | write branches + update frontmatter |
| `bo query` | select relevant context (+ optional cheap ranking call) | synthesize answer | output / write result file |
| `bo lint` | gather index + branch summaries | identify problems / suggestions | report |

## Architectural insight: bo is a primitive, not an orchestrator

Bo's CLI commands become reliable, composable tools that external agents can invoke.
The layering is:

```
┌─────────────────────────────────────────────────────┐
│  Agent layer  (NOT bo's concern)                    │
│  Claude Code, pi, Cursor, custom agents             │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│  Interface layer                                    │
│  CLI (humans + scripts)  /  MCP server (agents)     │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│  Bo core                                            │
│  Deterministic pipelines with structured LLM calls  │
└─────────────────────────────────────────────────────┘
```

Bo doesn't need to build or maintain an orchestration layer. Anyone with an agent
setup gets full automation of their knowledge base by adding bo commands to their
tool list — agentic capability for free.

## Practical implications

- Remove the internal agent loop from `compile` (`engine/agent/`)
- Replace with a single structured-output LLM call
- Keep `LlmProvider` trait for provider abstraction
- CLI output should support `--json` for machine consumption
- ~20× fewer API calls per compile → dramatically cheaper
- No step-limit ceiling on collection size
- Fully testable: mock the one LLM call, test everything else deterministically
