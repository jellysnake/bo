### 2. Zero-citation = not-answered

**Goal:** If query synthesis produces zero valid citations, treat the result as unanswered rather than emitting a hallucinated response.

- [x] After synthesis, check `cited_slugs`
- [x] If empty: human output "no answer from collected sources", exit 1; JSON `insufficient_sources`
- [x] Unit tests: zero-citation → not-answered, ≥1 citation → normal
- [x] Integration test: query against irrelevant tree → exit 1

---

### 3. `bo config set` MVP

**Goal:** Users can change the model without hand-editing JSON.

- [x] Single `model` field for v0.0.1. One knob, affects all LLM stages.
- [x] `bo config set model <value>` / `bo config get model`
- [x] Unknown key → exit 2 with error listing valid keys
- [x] `--json` support

---

### 4. Clean installation happy path

**Goal:** New user can install bo from GitHub without cloning the repo.

- [x] Document: `cargo install --git https://github.com/jellysnake/bo --tag v0.0.1`
- [x] Installed `bo` works from any local shell / arbitrary working directory
- [x] Smoke check validates: `bo --help`, `bo config get model`, `bo seed <tree-dir>`, `bo list`

---

### 5. Auth configuration happy path

**Goal:** Users configure their OpenAI API key through bo itself.

- [x] `bo config auth --provider openai` prompts for key without echoing
- [x] Writes `~/.bo/auth.json` with restrictive permissions (`0600`)
- [x] Resolution order: `OPENAI_API_KEY` env → `~/.bo/auth.json` → clear error
- [x] Error messages point to `bo config auth --provider openai`
- [x] JSON mode works, never includes secret value
